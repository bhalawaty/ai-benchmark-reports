How Tokens Work — Simple Explanation

Tokens are the "currency" of AI models. Think of them like data credits — every word, number, and symbol costs tokens.

💬
User Asks a Question
"Show me employees who were late this month"
~50 tokens
🔧
System Adds Context
Database schema, rules, instructions sent with every request
~2,500 tokens
🤖
AI Returns SQL
SQL query + brief explanation
~1,200 tokens

Why the system prompt is included every time: Unlike a chat where previous context is "remembered," each AI SQL request sends the full database schema and rules with the question. This is ~2,500 tokens of input on every single request — it's the biggest cost driver.

Token Subscription Model — How It Works

Each company subscribes to a token quota — like buying a data plan. They don't pay per-month, they pay for a bucket of tokens that gets used up as employees ask questions.

💳
Company Buys Tokens
e.g., "1M token plan"
Like buying a prepaid card
👥
Employees Use Chat
Each question uses ~4,000 tokens
Shared across all admins
🚫
Quota Runs Out
Company upgrades or waits
We set limits per plan
Why token-based (not monthly)? A company with 5 admins asking 10 questions/day uses very differently from one with 50 admins asking 100 questions/day. Token quotas let us charge fairly based on actual usage — like cloud bandwidth.

Pricing Strategy Insight

Cheapest Option: Gemini 2.5 Flash

At $0.99/M blended cost, a 1M token plan costs us less than $1. Even a generous 5M plan costs only $4.95. This gives the widest margin for subscription pricing.

Most Expensive: Claude Opus 4.6

At $11.00/M blended cost, a 1M plan costs us $11.00. A 5M plan = $55. This model is only viable for premium tiers or internal use — not volume SaaS.

Recommendation for SaaS pricing: Use Gemini 2.5 Flash (or Pro after fine-tuning) as the default model. Offer Claude Opus as a "Premium AI" tier at higher subscription prices. This way our cost basis stays low while offering customers a choice.
